# Github pages

En esta sección aprenderemos a deployar una página (o sitio web) estática mediante github pages.

# Página

Como ejemplo, partiremos de algún jupyter notebook .ipynb estudiado en clase, y lo exportaremos a .html (File/Export/HTML). También podríamos utilizar un archivo RMarkdown, o un sitio entero hecho con Rbookdown, o cualquier otra tecnología que construya un sitio web estático (por ejemplo vuepress).

Nombraremos nuestra página index.html.

# Github

Si aún no lo hemos hecho, nos crearemos una cuenta en github (opens new window). Github nos permite compartir nuestro código fuente, proyectos, aplicaciones de código abierto, mantener un control de las versiones, resolver los problemas en nuestras librerías con ayuda de la comunidad, utilizar CI (continuos integration) con Travis-CI, o compartir páginas web de código abierto, entre otras muchas cosas.

Una vez en nuestra cuenta, iremos a https://github.com/new y crearemos un nuevo repositorio. Escogemos el nombre, podemos añadir una descripción, e iniciaremos el repostorio con un archivo README.md.

Luego de crear el repositorio, subiremos nuestro archivo index.html y haremos commit.

Finalmente, iremos a Configuración (Settings/Options) y ahí bajaremos hasta la opción GitHub Pages donde activaremos que el repositorio será publicado en https://username.github.io/repo/

¡Eso es todo!
