# KMeans

K-means es, probablemente, uno de los algoritmos de clúster más útiles y eficientes computacionalmente. Sin embargo, tiene dos desventajas fundamentales.

  1. Se considera únicamente la distancia euclídea.
  2. Hay que especificar desde el comienzo el número de clusters que queremos.

El funcionamiento de este algoritmo se ilustra aquí (opens new window).

Para hacer KMeans en python utilizamos la clase KMeans de la librería sklearn.

from sklearn.cluster import KMeans
import numpy as np

Comenzamos ajustando un modelo KMeans con dos grupos.

kmeans = KMeans(n_clusters=2, random_state=0).fit(data_protein)

kmeans.labels_

# Preprocesado

Es importante tener en cuenta que, al KMeans estar basado directamente en la distancia euclídea (estrictamente, el cuadrado de la distancia euclídea), este algoritmo tiene algunas características.

  1. Todas las variables en los datos deben ser de tipo numérico. Si hay alguna variable categórica, debe antes codificarse apropiadamente (por ejemplo, como variables dummies 0-1). No obstante, incluir variables binarias puede provocar agrupaciones triviales.

  2. KMeans es muy sensible a outliers. Debido a que la medida utilizada para cuantificar dis-similiaridad entre observaciones (que no es una distancia en el sentido formal) es más sensible que la distancia euclídea, esto hace que KMeans le asigne mucho peso a observaciones distantes.

  3. Todas las columnas deben tener varianzas similares, de lo contrario KMeans tenderá a separar los clusters por la variable con mayor varianza. A menos que la aplicación lo impida (por ejemplo, matrices sparse), siempre se debe escalar los datos para tener la misma varianza en todas las columnas, y media cero, antes de ajustar KMeans.

from sklearn import preprocessing

scaler = preprocessing.StandardScaler().fit(data_protein)
df_scaled = scaler.transform(data_protein)

Ahora que hemos escalado los datos, podemos hacer KMeans. Esta vez, calcularemos tres grupos.

km = KMeans(n_clusters=3, random_state=0).fit(df_scaled)

Vamos a añadir la información de los clusters en data_protein y además vamos a incluir la transformación en los datos para poderlos representar en un gráfico.

data_protein['cluster'] = km.labels_
data_protein[['x1', 'x2', 'x3']] = km.transform(df_scaled)

# Selección del número de clusters

En esta sección veremos dos técnicas para seleccionar el número óptimo de clústers: el método del codo y el método de la silueta.

# Método del codo

La idea básica de los algoritmos partitivos es obtener cluster con la mínima $wss$ (within-cluster sum of squares), que mide cuán compactos son los clusters. Pudiéramos intentar quedarnos con el número de clusters que minimiza este valor. Sin embargo, $wss$ siempre decrece a medida que consideramos un mayor número de grupos.

El método del codo mira el valor de $wss$ con respecto al número de grupos considerados, y busca el primer punto en que hay un cambio brusco en la curva, es decir, que adicionar un grupo nuevo no mejora demasiado con respecto a lo que ya había.

km = KMeans(n_clusters=2, random_state=0).fit(df_scaled)
km.inertia_

139.5073704483181

km = KMeans(n_clusters=3, random_state=0).fit(df_scaled)
km.inertia_

110.40242709032154

km = KMeans(n_clusters=4, random_state=0).fit(df_scaled)
km.inertia_

90.41954159596904

La librería yellowbricks incluye la funcionalidad para visualizar el método del codo.

# Silueta

El análisis de la silueta se puede utilizar para estudiar la distancia de separación entre los clústers resultantes. La visualización de la silueta muestra una medida de cuán cercano está cada punto de un cluster a puntos de sus clusters vecinos. El método de la silueta, para cada agrupación produce un score entre 1 y -1, donde 1 significa que hemos encontrado clusters muy densos, y -1 una agrupación totalmente incorrecta. Valores cercanos a cero indican que dicha observación se encuentra en el límite de otro cluster.

from yellowbrick.cluster import SilhouetteVisualizer

km = KMeans(6, random_state=0)
visualizer = SilhouetteVisualizer(km).fit(df_scaled)


km = KMeans(5, random_state=0)
visualizer = SilhouetteVisualizer(km).fit(df_scaled)


km = KMeans(4, random_state=0)
visualizer = SilhouetteVisualizer(km).fit(df_scaled)


km = KMeans(3, random_state=0)
visualizer = SilhouetteVisualizer(km).fit(df_scaled)
